Kinaxis License Access
Process Improvement

Supply planners at Diamond Pharmaceuticals need Kinaxis software licenses to execute their planning activities. Today, granting or modifying those licenses is a slow, manual process that delays work, increases costs, and creates audit risks. This project eliminates those pain points by automating the request, approval, and provisioning workflow end-to-end.

Current status — September 1, 2026
Risk #4 hit: Workflow actions not synchronized
During the August 31 pilot, a requester reported that an automated request submitted about a week earlier still had no update. The team reviewed the automation log and found that the approver's decision had been made, but the workflow did not capture it correctly — so the requester was never notified and the license was never granted.
Mitigation already in place before the risk hit
The team had already planned to test and validate the workflow before and during the pilot phase — as documented in the Risk Management Plan: "Conduct workflow testing and validation to ensure approval decisions flow correctly." This prior effort enabled the team to quickly identify the exact failure point in the automation log.
Contingency response — how the team reacted
The automated workflow was paused until the issue could be corrected
Requests are being processed manually so that license access can continue without further delay
The team is working with IT to fix the workflow logic before restarting the automated process — estimated to take more than 15 days
The affected pilot user was notified directly and their license was manually granted
Project impact
The launch milestone is delayed — new date to be determined after meeting with IT to identify and resolve the workflow bug

Frequently asked questions
How will user data be managed and protected?
All license request data will be stored in a cloud-based centralized repository hosted on AWS. Access is controlled through AWS IAM (Identity and Access Management), which restricts repository access to specific authorized team members only — including the COE License Processor, Kinaxis Admin, Request Approver, Technical Manager, Audit Expert, and Regulatory/Finance team.
When will the project be completed?
The final milestone (user satisfaction survey) is now planned for January 15, 2027, revised from December 15, 2026 due to the workflow issue identified during the pilot phase. The go-live launch date is to be determined after the IT fix is completed and retested.
How will this project benefit the company?
Supply planners will receive access in 4 business days instead of 7, allowing them to start working sooner. Processing effort per user drops from 5 hours to 30 minutes, and the company can expect to save $5,000 per week by reducing delays. The total cost per user will drop from $2,951 to $1,701 — a 42% reduction. The automated workflow also reduces human error and maintains a complete, audit-ready record of every access decision.
Where can stakeholders go with questions?
All questions and concerns should be directed to Project Manager Daniela Ortiz. Urgent issues can be escalated to Project Sponsor Mike Day. The project team holds weekly status meetings and publishes summaries after each major milestone.
